NodeJS endpoints

Endpoints

  • /getImage

    GET request to the server, sending a product id. The server gets the products with that ad and gets the corresponding image from /back/uploads.

  • /getAllCategories

    GET request to the server, sending a people type(men/women/girl/boy). The server calls the getCategories() function from db.js with that type and gets the categories array which sends back to front.

  • /getCartProducts

    GET request to the server, sending a user email. The server calls the getCart() function from db.js and gets cart products array for that user.

  • getAllProductsForCategories

    GET request to the server, sending a category name and a people type. The server calls the getProductsByPeopleAndCategory() function from db.js and returns the products array.

  • /getAllProducts

    GET request to the server. The server calls the getProducts() function from db.js and returns the products array.

  • /getAllOrders

    GET request to the server. The server calls the getAllOrders() function from db.js and returns the orders array.

  • /getProductData

    GET request to the server, sending a product name. The server calls the findProduct() function from db.js and returns the product as object.

  • /getAllUsers

    GET request to the server. The server calls the getUsers() function from db.js and returns the users array.

  • /getOrders

    GET request to the server, sending a user email. The server calls the getOrders() function from db.js and returns the orders array for that user.

  • /getUserData

    GET request to the server, sending a user email. The server calls the findUser() function from db.js and returns the user as object.

  • /getFavorites

    GET request to the server, sending a user email. The server calls the getFavorites() function from db.js and returns the favorites array for that user.

  • /getAddresses

    GET request to the server, sending a user email. The server calls the getAddresses() function from db.js and returns the addresses array for that user.

  • /addUser

    POST request to the server, sending a user object. The servers adds the user to the users collection by calling the createUser() function from db.js.

  • /addProductToCart

    POST request to the server, sending a user email, product name and selected color and size for that product. The servers adds the product to the carts collection by calling the addProductToCart() function from db.js.

  • /addProductToFavorites

    POST request to the server, sending a user email, product name and selected color and size for that product. The servers adds the product to the favorites collection by calling the addProductToFavorites() function from db.js.

  • /addProductToCart

    POST request to the server, sending a user email, product name and selected color and size for that product. The servers adds the product to the carts collection by calling the addProductToCart() function from db.js.

  • /addProductFromCategoriesToCart

    POST request to the server, sending a user email, product name and selected color and size for that product. The servers adds the product to the carts collection by calling the addProductToCart() function from db.js.

  • /addProduct

    POST request to the server, sending a user product as object. The servers adds the product to the products collection by calling the createProduct() function from db.js.

  • /updateProductImage

    POST request to the server, sending product name and image. The servers updates the product image in products collection by calling the updateProductImage() function from db.js.

  • /updateProduct

    POST request to the server, sending product name and changes for that product. The servers updates the product in products collection by calling the updateProductNoImage() function from db.js.

  • /updateUser

    POST request to the server, sending a user email and changes for that user. The servers updates the user in users collection by calling the updateUser() function from db.js.

  • /updateAddress

    POST request to the server, sending a user email, an address name and changes for that address. The servers updates the address in addresses collection by calling the updateAddress() function from db.js.

  • /checkUser

    POST request to the server, sending a user email and password. The servers checks if the user with that email and password exists by calling the findUser() function from db.js.

  • /resetPassword

    POST request to the server, sending a user email and a new password. The servers checks if the user exists and changes the password by calling the resetPassword() function from db.js.

  • /addNewOrder

    POST request to the server, sending a user email and a order as object. The servers adds that order for that user in the orders collection by calling the addOrder() function from db.js

  • /addNewAddress

    POST request to the server, sending a user email and a address as object. The servers adds that address for that user in the addresses collection by calling the addAddress() function from db.js

  • /deleteProductFromFavorites

    DELETE request to the server, sending a user email and a product to delete from favorites. The server deletes that product for that user from the favorites collection by calling the deleteFromFavorites() function from db.js.

  • /deleteProductFromCart

    DELETE request to the server, sending a user email and a product to delete from cart. The server deletes that product for that user from the carts collection by calling the deleteFromCart() function from db.js.

  • /deleteProduct

    DELETE request to the server, sending a product name to delete that product from the db. The server deletes that product from the products collection by calling the deleteProduct() function from db.js.

  • /deleteUser

    DELETE request to the server, sending a user email to delete that user from the db. The server deletes that user from the users collection by calling the deleteUser() function from db.js.
